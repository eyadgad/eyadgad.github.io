1.1 Problem 1


Vecna wants to develop a grade management system for his students to store their records. He has records and each record is a list consists of a student's name, course name, and associated grades. The student's name and course name are strings, while the grades are represented as a tuple containing coursework and final exam grades(coursework, final exam), with their total not exceeding 100.
For example, in the input of test case 1, a record might look like ['Sara', 'English', (60, 40)], indicating Sara got full marks in both coursework and the final exam for the English course. As students may enroll in multiple courses, their names could appear multiple times in the records. In the same example, if Sara also takes a French course and receives grades (35, 38),there would be two records for Sara.

Vecna wants to transform these records and add them into a new list where each element is a list represents a unique student along with their enrolled courses and corresponding grades. For example, in the output of test case 1, Sara's transformed record would be ['Sara', ['English', 'French'], [(60, 40), (35, 38)]]. In this format, the first element is the student's name, the second element is a list of the courses taken, and the third element is list of tuples representing the grades for each respective course.

Standard Input

  • A single integer number representing the number of records.
  • Multiple lines, each line contains student's name, course name, and associated grades separated by two commas.

    • Standard Output

  • Print the elements of the new list, each element is a single list representing unique student along with their courses and corresponding grades.

    • Test Case 1

    Input Sequence

    2↵Sara,,English,,(60,40)↵Sara,,French,,(35,38)↵

    Output Sequence

                ['Sara', ['English','French'], [(60,40),(35,38)]]

    1.2 Problem 2


    Vecna now wishes to enhance the grade management system by incorporating a feature to calculate the overall GPA for students based on their records. A record represents a unique student along with their courses and corresponding grades. The overall GPA is calculated using a specific grading scale where each grade corresponds to a Grade Point (GP) value. The GP values range from 4.0 GP for grades between 100-95 marks to 0 GP for grades below 50 marks, as outlined in the table below:

    GPGrades
    4.0100 - 95
    3.794 - 90
    3.389 - 85
    3.084 - 80
    2.779 - 75
    2.374 - 70
    2.069 - 65
    1.764 - 60
    1.359 - 55
    1.054 - 50
    0.0Below 50

    To calculate the GPA for a student:
  • The GP value for each course is determined by mapping its final grade to the corresponding GP value from the table.
  • The GPA is then calculated by averaging the GP values.
    • As an example in test case 1, for a student named Jane with grades of 90 in French and 85 in Math, her respective GP values would be 3.9 and 3.8. By averaging these GP values, Jane's overall GPA is calculated as (3.9 + 3.8) / 2 = 3.85

    In this problem, You are provided with unique records and each record represents a unique student along with student's enrolled courses and corresponding grades. Your task is to calculate the overall GPA for each student and print the top 5 students based on their GPAs.

    Standard Input

  • A single integer number representing the number of unique records.
  • Multiple lines, each line is a record consists of student's name, courses, and associated grades separated by two commas.

    Standard Output

  • Print the top 5 students in descending order based on their GPAs. Each line contains student's name and their GPA separated by a comma, as "{student's name},{GPA}".

    Test Case 1

    Input Sequence

    6↵Jane,,['French','Math'],,[(56,34),(55,30)]↵Mike,,['English','Physics','French','Math'],,[(50,30),(42,20),(10,28),(12,25)]↵Justin,,['English','Chemistry','Physics','Math'],,[(60,38),(30,28),(32,15),(29,25)]↵Max,,['History'],,[(51,35)]↵Steve,,['English','Chemistry','Physics','Math'],,[(29,28),(23,28),(22,28),(23,15)]↵Will,,['Chemistry','Physics','Math'],,[(60,39),(59,38),(57,37)]↵

    • 1.3 Problem 3


    In the quest to foster an equitable grading system and ensure a high standard of education, Vecna has introduced a universal passing criterion applicable across all courses. According to this new regulation, a student will successfully pass a course only if they achieve a minimum of 60% in coursework and a minimum of 50% in the final exam. There are some exceptions that students may be did not take the final exam due to medical reasons. In this case, the final exam grade will be -1 and the final grade will be incomplete. Vecna now seeks to integrate this universal passing criterion into the existing grade management system to automatically determine the pass(P) or fail(F) or incomplete(INC) status for each student based on the transformed records where each record represents a unique student along with their courses and corresponding grades.

    For example, in the first test case, Sara's courses and their respective grades. After applying the universal passing criterion, it is determined that Sara will pass in English and Physics. However, she will not pass in Math due to a final exam grade below the 50% threshold. Additionally, as Sara did not take the final exam for French, her status for that course is incomplete.

    Standard Input

  • A single line containing a record consists of student's name, courses, and associated grades separated by two commas.

    • Standard Output

  • A single list containing the P or F or INC status for each course respectively.

    • Test Case 1

    Input Sequence

    Sara,,['English','French','Physics','Math'],,[(60,40),(35,-1),(52,20),(55,10)]↵

    Output Sequence

                ['P','INC','P','F']

    Test Case 2

    Input Sequence

    Dan,,['Math','English','French','Physics'],,[(54,30),(50,39),(15,34),(42,15)]↵

    Output Sequence

                ['P','P','F','F']

    Test Case 3

    Input Sequence

    Jane,,['French','Math'],,[(56,34),(55,30)]↵

    Output Sequence

                ['P','P']
